# Shayfa Pro - Production Deployment Guide

This document contains step-by-step instructions to deploy the production-ready Quran platform.

## 1. Backend (The Engine)
- **Local Run**: `cd backend && npm install && npm run dev`
- **Database**: Create a MongoDB Cluster on [MongoDB Atlas](https://www.mongodb.com/cloud/atlas).
- **Storage**: Create a Firebase project, enable "Storage", and download the `serviceAccountKey.json`.
- **Cloud Deploy**: Use [Railway.app](https://railway.app/) or [Render.com](https://render.com/).
  - Connect your GitHub repo.
  - Add all variables from `.env.example` to the platform's Environment Settings.

## 2. Admin Panel (The Workspace)
- **Local Run**: `cd admin-panel && npm install && npm run dev`
- **Build**: `npm run build`
- **Deploy**: Drop the `dist` folder into [Netlify] or [Vercel].
- **Security**: Ensure you update `API_BASE` in `Dashboard.jsx` to your live Backend URL.

## 3. Mobile App (The User Experience)
- **Setup**: Install [Flutter SDK](https://docs.flutter.dev/get-started/install).
- **Config**: Update `baseUrl` in `lib/api_service.dart` to your live Backend URL.
- **Build APK**: `flutter build apk --release`
- **Build AAB (Play Store)**: `flutter build appbundle`
- **Play Store Requirements**:
  - Target SDK: 33+
  - Permissions: Internet, Write External Storage (for offline).

## 4. Google Play Console Listing
- Generate icons using `flutter_launcher_icons`.
- Prepare Screenshots (6.5", 5.5" and Tablet).
- Upload the `.aab` file from `build/app/outputs/bundle/release/app-release.aab`.
